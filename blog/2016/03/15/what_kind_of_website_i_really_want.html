写在开始

Mar 15, 2016 • 郭晓灰灰


其实这篇文章是写于2016-04-02日，而非2016-03-15，我将它放在2016-03-15的原因是为了纪念这一天我拿到了我梦想已久的驾照。一扫我在2016-01-27科目二考试失利的感觉，讲真，我从来不是一个非常自信的人，我的自信从暑假实习以来都在一点一滴的流失，我再也回不到过去那种对自己内心狂热的自信。这种自信流失终于终止于2016-03-15，我是在2016-02-29补考的科目二，在2016-03-07一把过的科目三，在2016-03-15满分考过科目四，拿到的驾照。从这一天开始，我不仅开始重塑我的自信，而且我有了更多的时间来学习。

大概是从今年的二月份开始，我接触到一个很吊炸天的名词markdown，当时我正好看完了一本名为《全栈工程师的自我修养》，作者说他的书是用markdown写出来的。当时只是想要耍帅的学习这个东西来写出不同的格式来亮瞎人们的双眼，但是随着接触的深入，我开始由学习markdown变成了学习怎么使用Jekyll来搭建自己的github pages，然后丢下了毕业设计，开了支线当主任务，所以每周开会的时候，总是一种虽然我什么都没有做，但是还是有一种神一般的从容和淡定的豪迈情怀。

差不多在这半个月中我接触了以下几个神器

1. atom 编辑器  
2. github 桌面版 版本管理工具  
3. Jekyll 静态博客开发工具  
4. Ruby

学习的内容有：

1. git版本控制工具的使用，git shell 和github桌面版  
2. SCSS、CSS、HTML5的进一步学习  
3. Jekyll博客工具和插件的基本使用  
4. Ruby语言的初级接触  
5. atom工具的使用和熟悉  
6. chrome浏览器在开发中的使用  
7. CSSViewer的使用


2016-04-02-想法

github pages应当有以下功能

1. 分页
2. 图片大小调整
3. 搜索功能
4. 音乐播放功能
5. 标签显示
6. 个人联系方式logo制作
